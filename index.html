The Problem: Typical LLM-Based Control Loop

The standard LLM-based robot planning and control loop is straightforward but suffers from fundamental limitations that prevent real-time, adaptive robot behavior:

  • High latency: The robot must wait for a full LLM inference cycle before executing any action, making real-time responsiveness difficult.
  • Blocking execution: The entire pipeline stalls during LLM inference — the robot cannot react to new observations or interrupts mid-cycle.
  • No speculative execution: Actions cannot be pre-fetched or pipelined; the robot remains idle while the model is thinking.
  • Sequential, single-threaded planning: Only one planning request is processed at a time, preventing concurrent handling of multiple tasks or goals.
  • Context window bottleneck: All observations, memory, and system descriptions must be packed into a single LLM prompt, limiting the richness of state that can be represented.

Figure 1 — Typical LLM-based robot planning and control loop

Our Solution: Asynchronous Multi-Agent Architecture

Each agent runs at its own independent update frequency, directly addressing every limitation of the typical loop.

Low-Latency Action Execution

Reactive agents operate at high frequency without waiting for slow LLM inference, enabling immediate responses to environmental changes.

Non-Blocking Pipeline

Agents run concurrently — observation, planning, and execution proceed in parallel rather than sequentially, keeping the robot continuously active.

Speculative Action Pipelining

The planner speculatively issues upcoming actions ahead of time, so the robot executes them as soon as the current action completes with minimal idle time.

Concurrent Multi-Task Handling

Independent agents manage multiple goals simultaneously, with a priority-based scheduler arbitrating between them in real time.

Distributed Context Management

Each agent maintains its own focused context, avoiding the single-prompt bottleneck and allowing richer, more specialized representations of state.
